# copy-cat

macOS screenshots that go **both** places at once:

- **Saved** to `~/Desktop/Screenshots` — a permanent historical record of everything you've captured.
- **Copied** to the clipboard — ready to paste immediately.

macOS itself only lets you pick one (the Cmd+Shift+5 → Options menu is clipboard *or* a folder, never both). copy-cat bridges the gap with a tiny background agent.

## How it works

1. Native screen capture (Cmd+Shift+3/4/5) is set back to **save to a folder** (`~/Desktop/Screenshots`).
2. A launchd `LaunchAgent` watches that folder (`WatchPaths`).
3. When a new screenshot lands, it copies the image to the clipboard via `osascript`.

No third-party app, no Screen Recording permission, no rebinding the system shortcuts. Just a script + a plist.

## Install

```bash
./install.sh
```

Then test: Cmd+Shift+4, grab a region, and Cmd+V somewhere. The file is in `~/Desktop/Screenshots` and the image is on your clipboard.

## Other laptops

Clone the repo and run `./install.sh`. The installer generates the LaunchAgent with the correct paths for whatever user/home it runs under, so it's portable as-is.

## Uninstall

```bash
./uninstall.sh
```

Leaves your screenshots folder and its contents untouched.

## Files

| Path | Purpose |
|---|---|
| `screenshot-to-clipboard.sh` | The helper: copies the newest screenshot to the clipboard. Installed to `~/.local/bin/`. |
| `install.sh` | Sets the screenshot save location, installs the helper, generates + loads the LaunchAgent. |
| `uninstall.sh` | Unloads the agent and removes installed files. |
| `~/Library/LaunchAgents/com.0x63616c.screenshot-clipboard.plist` | Generated by the installer. Watches the folder. |
| `~/.local/state/screenshot-clipboard.log` | Agent stdout/stderr, for debugging. |

## Notes

- `ThrottleInterval` is set to `1` so rapid-fire screenshots aren't delayed by launchd's default 10s relaunch throttle.
- The helper dedupes on file path + mtime, so folder changes from *deleting* a screenshot don't re-copy an older one.
